# 技术设计说明（Technical Design）

## 1. 设计目标
围绕“广州 + AI产品经理”固定口径，设计一套可稳定运行的岗位数据管线，满足以下目标：
- Playwright 作为主采集器。
- MCP 作为增强层，非阻塞接入。
- 在反爬和网络波动场景下具备容错与降级能力。
- 标准化输出数据与报告，支持定时执行。

## 2. 总体架构
采用分层架构：

1. 采集层（Collection）
- 模块：`playwright_collector.py`
- 职责：在线采集 51job 岗位数据，处理分页、反爬、请求重试。

2. 处理层（Processing）
- 模块：`processing.py`
- 职责：清洗、去重、薪资标准化、剔除实习岗位、统计预处理。

3. 增强层（Enhancement）
- 模块：`mcp_enhance.py` + `mcp_adapter.py`
- 职责：对最新样本补充技能和 AI 标签等增强信息。

4. 输出层（Reporting）
- 模块：`reporting.py`
- 职责：生成 CSV 与 Markdown 报告。

5. 编排层（Orchestration）
- 模块：`main.py`
- 职责：串联全流程，管理参数、路径、执行顺序和失败降级。

## 3. 主流程设计
主流程：
1. `collect_jobs`：优先 Playwright 在线采集。
2. 若在线失败：进入降级路径（如快照解析）。
3. `process`：标准化字段并过滤实习岗位。
4. `enhance_jobs`：对有限样本调用 MCP 增强。
5. `generate_report`：输出 `jobs.csv` 与 `Report_YYYYMMDD.md`。

## 4. 采集层详细设计

### 4.1 分页策略
- 基于接口返回的总量/分页信息计算目标页。
- 支持配置 `max_pages`，并采用安全上限避免异常无限抓取。
- 去重键建议使用 `job_title + company_name + city`（或 URL）。

### 4.2 反爬机制
- 随机等待与节流，降低请求特征。
- 识别挑战页特征（如 JS challenge 标识）并切换策略。
- 使用浏览器上下文发起请求，复用会话 Cookie。
- 重试机制：指数退避或固定次数重试。

### 4.3 降级策略
- 在线采集失败时回退到本地快照文件。
- 快照解析需兼容 51job 实际结构（如 `resultbody.job.items`）。
- 降级来源写入元数据字段，便于追溯。

## 5. 处理层详细设计

### 5.1 字段标准化
- 统一输出核心字段：职位、公司、城市、薪资原文、发布时间、链接等。

### 5.2 薪资解析
- 支持月薪、年薪、含薪数等常见表达。
- 统一转换到 `K/月`。
- 产出：`salary_min`、`salary_max`、`salary_avg`。

### 5.3 业务过滤
- 剔除标题或描述中包含“实习/intern”的岗位。
- 过滤后数据作为统计和 Top10 的数据源。

## 6. 增强层详细设计
- 通过 `mcp_adapter.py` 适配外部服务或本地 stub。
- 控制增强规模（如最新 5 条）以平衡时效与成本。
- 超时/失败时标记 `enhance_source`，不中断主流程。

## 7. 报表层设计

### 7.1 产物规范
- 数据文件：`04AI_Job_Report/jobs.csv`
- 报告文件：`04AI_Job_Report/Report_YYYYMMDD.md`

### 7.2 报告内容
- 市场仪表盘：岗位总数、薪资中位数、查询口径。
- 薪资分布：分箱统计与图示。
- Top10 最新岗位：按发布时间排序。
- AI 增强摘要：技能、AI 占比、工具等。

## 8. 运行与调度设计
- 本地执行：`python main.py`（支持参数覆盖默认关键词和分页）。
- 定时调度：通过 GitHub Actions 或 cron 模板按周执行。
- 运行失败时输出错误日志，便于快速定位。

## 9. 关键配置项
- `keyword`：默认 `AI产品经理`。
- `city`：默认 `广州`。
- `max_pages`：分页上限。
- `mcp endpoint`：外部增强服务地址（可选）。
- `output_dir`：默认 `04AI_Job_Report/`。

## 10. 质量与验收
- 功能验收：全链路可运行并输出两类产物。
- 数据验收：分页有效，结果不含实习岗位。
- 稳定性验收：在线失败时可降级并产出结果。
- 一致性验收：关键词与报告命名规则统一。

## 11. 风险与后续优化
- 风险：站点接口变动、反爬升级、样本偏差。
- 优化方向：
	- 多站点扩展与统一抽象。
	- 更精细的相关性过滤与去重策略。
	- 增强层并发控制与缓存复用。
