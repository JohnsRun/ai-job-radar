# 业务需求说明（Business Requirement）

## 1. 项目目标
构建一个面向招聘市场洞察的轻量级自动化系统，以前程无忧（51job）为首要数据源，按固定口径持续采集并分析岗位数据，输出结构化数据和可读报告，支持周期性自动运行。

## 2. 固定业务口径（必须）
- 城市：广州。
- 关键词：AI产品经理（查询口径可表达为 `AI产品经理+广州`）。
- 报告关注对象：与 AI + 产品经理相关岗位。
- 过滤规则：剔除“实习/Intern”类岗位。

## 3. 核心功能需求
1. 岗位采集
- 主采集器使用 Playwright，优先在线采集 51job 数据。
- 支持分页抓取，不得只采集第一页。
- 需具备反爬应对能力（挑战页识别、重试、延时、浏览器上下文请求等）。

2. 数据处理
- 对岗位数据进行清洗、去重、字段标准化。
- 薪资统一解析为 K/月，输出最小值、最大值、均值。
- 剔除实习类岗位后再进入统计流程。

3. AI 增强分析
- 增加 MCP 增强层，对最新岗位样本进行能力补充（技能、AI相关性、工具等）。
- 增强层失败不得阻断主流程。

4. 报告与产物
- 输出 CSV：岗位明细数据。
- 输出 Markdown 报告：市场概览、薪资分布、Top10 最新岗位、增强分析摘要。
- 报告输出目录统一为 `04AI_Job_Report/`。
- 报告命名规则：`Report_YYYYMMDD.md`。

5. 自动化运行
- 支持定时任务（周级别）运行，自动生成最新数据与报告。

## 4. 非功能需求
- 稳定性：采集失败时需自动降级（例如快照回退），避免全流程失败。
- 可维护性：模块化分层，主流程清晰，便于后续扩展到更多招聘站点。
- 可追溯性：报告中应体现查询口径与样本来源。

## 5. 输入输出定义
- 输入：关键词（默认 AI产品经理）、城市（默认广州）、最大分页数等运行参数。
- 输出：
	- `04AI_Job_Report/jobs.csv`
	- `04AI_Job_Report/Report_YYYYMMDD.md`

## 6. 验收标准
1. 能稳定运行主流程：采集 -> 清洗 -> 增强 -> 报告。
2. 分页采集生效，采集量明显高于单页样本。
3. 报告文件生成在指定目录并符合命名规范。
4. 结果中不包含实习岗位。
5. 文档与报告关键词口径统一为“AI产品经理”。
